## 🔹 Task 04 – פיצוח Hashים בסיסיים

### 🔍 מה עשיתי

* קיבלתי Hashים שונים (MD5, SHA1, SHA256, SHA512)
* זיהיתי את סוג ה־Hash לפי מבנה ואורך
* הרצתי פיצוח באמצעות John + wordlist

### ❓ למה עושים את זה

במקרה של פריצה למסד נתונים, הסיסמאות לא נשמרות כטקסט רגיל אלא כ־Hash.
אם הסיסמה חלשה – ניתן לפצח אותה **אופליין**, בלי גישה למערכת.

### 🧠 מה למדתי

* אורך ה־Hash נותן רמז חשוב לסוגו
* Hash ≠ הצפנה שניתן “לפענח” – אלא השוואה
* סיסמאות פשוטות נופלות במהירות

---

## 🔹 Task 05 – פיצוח NTLM (Windows)

### 🔍 מה עשיתי

* פיצחתי Hash מסוג NTLM
* השתמשתי ב־John במצב המתאים ל־Windows hashes

### ❓ למה זה חשוב

NTLM נמצא בשימוש נרחב ב־:

* Windows
* Active Directory
* ארגונים גדולים

פיצוח NTLM מאפשר:

* שחזור סיסמאות
* Pass-the-Hash attacks

### ✅ תוצאה

```
mushroom
```

---

## 🔹 Task 06 – פיצוח /etc/shadow (Linux)

### 🔍 מה עשיתי

* שילבתי את הקבצים `/etc/passwd` ו־`/etc/shadow` באמצעות `unshadow`
* פיצחתי סיסמת root

### ❓ למה זה קריטי

`/etc/shadow` מכיל את הסיסמאות המוצפנות של כל משתמשי המערכת.
גישה אליו = סכנה חמורה.

### ✅ תוצאה

```
1234
```

> דוגמה מצוינת לכך שגם Hash חזק (SHA512) **לא מציל סיסמה חלשה**.

## 🔹 Task 07 – Single Crack Mode

### 🔍 מה עשיתי

* ערכתי את קובץ ה־hash לפורמט: `username:hash`
* השתמשתי ב־`--single`

### ❓ למה זה עובד

משתמשים רבים בונים סיסמאות מ:

* שם משתמש
* וריאציות עם מספרים / אותיות גדולות

### ✅ תוצאה

```
Jok3r
```

## 🔹 Task 09 – קבצי ZIP מוצפנים

### 🔍 מה עשיתי

* פיצחתי סיסמה

### ❓ מתי זה קורה במציאות

* גיבויים מוצפנים
